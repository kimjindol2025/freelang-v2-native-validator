### 4개 통합 Phase

#### Phase 10: v1 Package 마이그레이션 (6주)

**목표**: v1의 31개 패키지 → v2 Intent 패턴 변환

**작업**:

1. **API 분석 및 Intent 매핑** (Week 1-2)
   ```
   v1 API → v2 Intent 패턴 변환 규칙

   예: http.request(...) → "HTTP GET 요청"
       fs.readFile(path) → "파일 읽기"
       crypto.hash(data) → "SHA256 해싱"
   ```

2. **패턴 데이터베이스 생성** (Week 3-4)
   - v1 패키지별 500+ Intent 패턴 생성
   - 각 패턴에 메타데이터 추가 (예: 신뢰도 초기값 = v1 테스트 통과율)
   - 카테고리 분류 (Core, Collections, Network, Security, Utils, DB)

3. **v1 코드 재사용** (Week 5-6)
   ```typescript
   // v1 코드 구조 유지
   src/
   ├── phase-10/
   │   ├── v1-patterns-migration.ts  (v1 API → Intent 패턴)
   │   ├── v1-packages-api.ts        (v1 패키지 소개)
   │   └── patterns-500+.ts          (500+ 새로운 패턴)
   └── vendors/
       └── freelang-v1/              (v1 코드 래퍼)
   ```

**파일**:
- `src/phase-10/v1-migration.ts` (1,500 LOC)
- `src/phase-10/patterns-500.ts` (3,000 LOC - v1 패턴)
- `docs/V1_INTEGRATION_GUIDE.md` (2,000 LOC)
- `tests/phase-10-v1-patterns.test.ts` (600 LOC)

**산출물**:
- 500+ 새로운 패턴
- v1 API 모두 Intent로 매핑
- 마이그레이션 문서

**신뢰도 계산** (v1 기반):
```
Pattern Confidence = v1_test_pass_rate × 0.9 + usage_frequency × 0.1
예: http(테스트 100% 통과) → confidence 0.95
    ws(테스트 100% 통과) → confidence 0.94
```

#### Phase 13: 커스텀 패턴 + v1 확장 (1주)

**목표**: 사용자/AI가 v1 패키지 기반 커스텀 패턴 정의

**작업**:

1. **패턴 정의 언어 (DSL)**
   ```
   @pattern(extends: "http")  // v1 http 패키지 기반
   name: custom_api_call
   input: string, object       // URL, 옵션
   output: object              // 응답
   description: "커스텀 API 호출"
   ```

2. **v1 패키지 확장 메커니즘**
   ```
   사용자가 정의 가능:
   - http.get() 기반 "JSON API 파싱"
   - fs.read() + json.parse() = "JSON 파일 읽기"
   - crypto 조합 = "파일 암호화"
   ```

3. **CLI 명령어**
   ```bash
   freelang --add-pattern custom_api.free
   freelang --list-patterns --filter "http"  # v1 http 패턴들
   freelang --export-patterns --package http # http 패키지 패턴 내보내기
   ```

**파일**:
- `src/phase-13/pattern-dsl-v1.ts` (700 LOC)
- `src/phase-13/v1-package-registry.ts` (500 LOC)
- `tests/phase-13-v1-patterns.test.ts` (500 LOC)

**성과**: v1 패키지를 기반으로 무제한 커스텀 패턴 가능
